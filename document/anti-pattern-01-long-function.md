# 너무 긴 함수 (Long Function)

## 개요
하나의 함수가 화면 렌더링, 데이터 조회, 검증, 로깅까지 모두 처리하는 패턴이다.
RAG로 코드 리뷰를 할 때 이 패턴을 발견하면 함수 분리(refactor)를 제안할 수 있다.

## 문제점
- 가독성이 떨어져서 신규 기여자가 진입하기 어렵다.
- 테스트를 작성하기 어렵다. (부분만 테스트하기 곤란)
- 변경 영향 범위를 예측하기 어렵다.

## 흔한 징후
- 함수 길이가 100줄 이상인데도 이름이 `process()` 나 `handle()` 처럼 추상적이다.
- 지역 변수가 5개 이상이고 계속 다시 할당된다.
- 주석으로 섹션을 나눠놓았다. (`// 1. validate`, `// 2. save` 등)

## 권장 리팩터링
- 역할 단위로 private 함수로 분리한다.
- 검증/변환 로직은 별도 모듈로 이동한다.
- 이름을 동사+목적어로 구체화한다. (예: `validatePayload`, `saveUserToDB`)
