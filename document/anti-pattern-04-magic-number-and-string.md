# 매직 넘버 / 매직 스트링 사용

## 설명
코드 안곳곳에 `"admin"`, `"PENDING"`, `3`, `5000` 같은 값을 그대로 넣는 패턴이다.
시간이 지나면 이 값이 왜 3인지, 왜 PENDING인지 설명할 근거가 사라진다.

## 문제
- 값이 변경될 때 전체 검색해서 바꿔야 한다.
- 리뷰어가 의도를 파악하기 어렵다.
- 국제화(i18n)나 설정화할 때 걸리적거린다.

## 예시
```ts
if (user.role === "admin") {
  retry(3);
}
```

## 개선
- 의미 있는 상수로 빼서 이름을 붙인다. (`ROLE_ADMIN`, `MAX_RETRY = 3`)
- 도메인에서 자주 쓰는 문자열은 enum이나 value object로 감싼다.
- 설정성 값은 환경변수나 설정 파일로 분리한다.
